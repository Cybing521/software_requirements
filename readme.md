# 软件需求 (Software Requirements) - 课程资料与大纲

本项目旨在整理软件需求工程课程的核心知识点、讲义内容及复习资料。本文档结构严谨地按照课程教学模块组织，既是项目概览，也是后续进行知识问答和复习的核心大纲。

## 项目大纲 (Project Outline)

本项目包含八个核心模块，涵盖了从需求基础到管理与实现的完整流程：

### 1. 基础概念 (The Four Ws)
*   **核心内容**: 定义软件需求，区分用户需求、业务需求、功能需求与非功能需求。
*   **关键文件**: `extracted_text/1-FourWs.txt`

### 2. 起始点 (Starting Point)
*   **核心内容**: 确立商业需求 (Business Requirements)，定义项目愿景与范围，理解业务目标。
*   **关键文件**: `extracted_text/2-StartingPoint.txt`

### 3. 需求获取 (Requirement Elicitation)
*   **核心内容**: 从利益相关者处挖掘需求。
*   **关键技术**: 面谈 (Interviews)、工作坊 (Workshops)、观察 (Observations)、问卷 (Questionnaires)、文档分析。
*   **关键文件**: `extracted_text/3-Elicitation.txt`

### 4. 需求分析 (Requirement Analysis)
*   **核心内容**: 对获取的需求进行建模(Modeling)和优先级排序(Prioritisation)。
*   **关键工具**: 用户故事 (User Stories)、用例 (Use Cases)、活动图 (Activity Diagrams)、实体关系图 (ERD)。
*   **关键文件**: `extracted_text/4-Analysis.txt`

### 5. 需求规格说明 (Requirement Specification)
*   **核心内容**: 编写软件需求规格说明书 (SRS)。
*   **关键要素**: 功能需求描述、数据字典、非功能需求 (质量属性：性能、安全性、可用性等)、外部接口。
*   **关键文件**: `extracted_text/5-Specification.txt`

### 6. 需求验证 (Requirement Validation)
*   **核心内容**: 确保需求的正确性和质量。
*   **关键活动**: 同行评审 (Peer Reviews)、原型设计 (Prototyping)、V模型验证 (测试对应关系)。
*   **关键文件**: `extracted_text/6-Validation.txt`

### 7. 需求管理 (Requirement Management)
*   **核心内容**: 应对需求变更，维护需求基线。
*   **关键实践**: 变更控制 (Change Control)、版本管理 (Version Control)、需求追踪 (Traceability)。
*   **关键文件**: `extracted_text/7-Management.txt`

### 8. 需求实现 (Requirement Implementation)
*   **核心内容**: 需求过程改进与最佳实践。
*   **关键文件**: `extracted_text/8-Implementation.txt`
